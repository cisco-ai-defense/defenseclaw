## Overview

`defenseclaw-gateway` is the Go sidecar daemon and runtime CLI. It connects to the OpenClaw gateway WebSocket, enforces policy on tool calls in real time, exposes the local REST API on port `18970`, serves the guardrail proxy on port `4000` when enabled, exports audit evidence, and runs CodeGuard source scans.

<Callout type="info">
  `defenseclaw tui` resolves this binary and execs `defenseclaw-gateway tui`.
  Daemon lifecycle commands are not Python wrappers; call `defenseclaw-gateway`
  directly for `start`, `stop`, `restart`, `status`, and `watchdog`.
</Callout>

## Running the sidecar

| Mode | Command | When to use it |
|------|---------|----------------|
| Foreground | `defenseclaw-gateway` | Local debugging, containers, and process managers that supervise PID 1. |
| Background daemon | `defenseclaw-gateway start` | Operator workstation or host install where DefenseClaw owns `gateway.pid` and `gateway.log`. |
| Graceful stop | `defenseclaw-gateway stop` | Stops the sidecar and watchdog path cleanly. |
| Restart after config/policy changes | `defenseclaw-gateway restart` | Reloads config read at startup and restarts the watchdog if configured. |
| Health check | `defenseclaw-gateway status` | Queries the running sidecar health endpoint and subsystem state. |

The background daemon writes `gateway.pid` and `gateway.log` under `~/.defenseclaw/`. If you changed config with `defenseclaw setup ...`, restart the sidecar so startup-read settings are reloaded.

## Day-two commands

```bash
defenseclaw-gateway status
defenseclaw-gateway policy reload
defenseclaw-gateway scan code ./src --json
defenseclaw-gateway audit export --limit 1000 --output audit.jsonl
defenseclaw-gateway watchdog status
```

<Callout type="warning" title="Avoid secrets on argv">
  The hidden `--token` flag still works for compatibility, but `internal/cli/sidecar.go` warns that command-line secrets are visible through process listings. Prefer `OPENCLAW_GATEWAY_TOKEN` or `gateway.token` in config.
</Callout>

## Persistent flags

`--host` and `--port` are root persistent flags. They override the gateway host and port used by commands that connect to the sidecar or start it. Most operators should leave them empty and let config drive the values.
