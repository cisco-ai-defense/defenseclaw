## Overview

`defenseclaw` is the Python operator CLI. It provisions config, drives skill/MCP/plugin scanners, manages credentials, edits policy, records operator activity, and launches the TUI by handing off to the Go gateway binary. Use it for decisions an operator owns; use `defenseclaw-gateway` for daemon lifecycle and sidecar-owned runtime operations.

<Callout type="tip">
  The Go gateway daemon is a sibling binary — see [Gateway CLI](/docs-site/cli/gateway-cli).
  The two CLIs are covered together in the [CLI overview](/docs-site/cli/index).
</Callout>

## Operator map

| Situation | Start with | Why |
|-----------|------------|-----|
| First install | `defenseclaw quickstart` | Creates config, bootstraps the audit DB, applies safe guardrail defaults, and can start the sidecar. |
| Controlled install | `defenseclaw init` | Gives you explicit scanner, sandbox, and guardrail setup choices. |
| Missing or broken config | `defenseclaw config validate`, `defenseclaw doctor` | These are recovery-safe commands and are not blocked by the normal config auto-validate hook. |
| Missing provider keys | `defenseclaw keys list`, `defenseclaw keys fill-missing` | Shows which keys are required by the current config and writes values to `~/.defenseclaw/.env`. |
| Skill/MCP/plugin review | `defenseclaw skill`, `defenseclaw mcp`, `defenseclaw plugin` | Lists, scans, blocks, allows, disables, quarantines, and restores local inventory. |
| Policy authoring | `defenseclaw policy` | Creates, validates, tests, activates, and edits local policy data. |
| Interactive operations | `defenseclaw tui` | Resolves and execs `defenseclaw-gateway tui` so the TUI uses the sibling gateway binary when available. |

<Callout type="info" title="Commands that work before full config load">
  `init`, `quickstart`, `sandbox`, `tui`, `uninstall`, `reset`, and `version` are allowed to run before normal config and audit-store loading. `config`, `keys`, `doctor`, `upgrade`, and `version` also skip the auto-validate lockout so operators can recover from a bad config file.
</Callout>

## Safe first commands

```bash
defenseclaw quickstart --mode observe --scanner local --skip-gateway
defenseclaw config validate --quiet
defenseclaw keys check
defenseclaw doctor --json-output
defenseclaw status
```

Start in `observe` mode when you are introducing DefenseClaw to an existing environment. Move to `action` mode only after alerts and scanner output match your expectations.
