## Overview

Use these journeys when you know your job but not yet the right page. Each path points into the same source-backed reference docs; the journey pages do not replace command references, policy specs, or generated API tables.

<Callout type="tip" title="Pick the closest role">
  If your team owns more than one responsibility, start with the role that matches today's task. Platform engineers usually start with installation and sidecar health, security teams with guardrail and policy, and developers with scanners and extension contracts.
</Callout>

## Choose a path

| Role | You are trying to | Start here | First verification |
|------|-------------------|------------|--------------------|
| Platform engineer | Install, configure, run, and expose DefenseClaw safely. | [Platform engineer](/docs-site/journeys/platform-engineer) | `defenseclaw doctor --json-output` and `defenseclaw-gateway status` |
| Security team | Roll out guardrails, policy, audit, alerting, and incident response. | [Security team](/docs-site/journeys/security-team) | `defenseclaw policy test --rego-dir policies/rego --verbose` |
